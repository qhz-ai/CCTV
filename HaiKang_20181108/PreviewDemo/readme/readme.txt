*主要阐述1，4，9画面轮询原理，和主要功能实现的原理以及开发中遇到的问题及解决办法。
*1，4，9画面的轮询：我将1画面的轮询和4，9画面的轮询做了分别的处理，主要是为了区别设置的多个标志位（容易搞混）。1画面的轮询在窗口刚生成的时候便开始进行轮询，轮询的时间
是程序右下角设置的时间间隔。默认是每隔15秒进行一次轮询。轮询规则是从程序左侧的第一个摄像头顺序进行轮询。中间我设置了一个标志位用来判断播放到第几个，当播放到最后一个的
时候，重新将标志位初始化。利用定时器进行轮询。4，9画面跟一个画面轮询最大的区别就是播放的画面要设置一个值进行自增，而1画面的轮询永远都是使用的第一个PictureBox.
*选中效果的实现：由于PictureBox显示视频流的特性，直接选中正在播放视频流的选中的效果会一闪而过，所以选择了一个折中的办法，即在播放视频的PictureBox后面再添加一个比播放视
频稍大一些的PictureBox,当选中正在播放的PictureBox时传值到后面的PictureBox，并添加选中效果。
*窗口正常<>最大化时播放窗口的自适应：原理很简单，就是将当前的PictureBox重新布局即可。
*摄像头轮询一段时间后出现画面缺失：是由于播放当前摄像头的时候，上一个视频流没有关闭，当没有关闭的视频流达到一定的数量以后便会出现画面缺失的情况。
*设备状态在线显示：通过判断设备的状态标志位来确定用具体的哪张图片来显示当前的状态。但只有在第一次连接摄像头的操作完成以后才会开始加载关于设备状态在线显示的图片。
*每隔一段时间都进行一次对初次连接失败的设备重新尝试连接，直到连接成功为止。当连接成功以后，隔一段时间，摄像头的状态便会改为在线状态。
*手动进行轮询设置的时候，无论是进行那个画面的轮询，对状态在线显示的定时器和轮询连接初次连接摄像头失败的定时器，都不会产生影响。
*初次连接摄像头的时候，利用线程池对所有摄像头的连接信息都存放到了一个list集合里，每一个摄像头又是一个list.需要对应摄像头的信息的时候直接从list取就行了。
*设置和调用功能的实现：利用海康提供的SDK即可快速的开发。
*鼠标双击切换1，4，9画面：双击当前的PictureBox的时候，会按照1-4-9画面进行轮询。同时会显示和当前画面数量相同的摄像头信息。默认是从第一个开始连接成功的摄像头来进行显示。
同时关于摄像头的轮询会停止。
